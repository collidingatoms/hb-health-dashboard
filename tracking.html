How to Export from Health Apps

Setup Guide

🍎 Apple Health (iPhone)

Option A — Full Export (Recommended for first-time)

  • Open the Health app on your iPhone
  • Tap your profile picture (top right)
  • Scroll down and tap "Export All Health Data"
  • This creates a ZIP file containing export.xml
  • AirDrop or email the ZIP to your computer
  • Extract the ZIP and upload the export.xml file here

Note: The full export can be large (100MB+). The parser on this page extracts only blood glucose and blood pressure records.

Option B — Use a Third-Party App for Selective Export

  • Health Auto Export (iOS, free tier): Exports specific data types as CSV. Set it to auto-export blood glucose and BP daily.
  • QS Access (iOS, free): One-tap CSV export of any Health data category
  • Export as CSV → upload directly to this page

🤖 Google Fit / Google Health Connect (Android)

Option A — Google Takeout (Bulk Export)

  • Go to takeout.google.com
  • Click "Deselect all", then check only "Fit" (or "Health Connect")
  • Click "Next step" → Choose "Export once" → "Create export"
  • Download the ZIP when ready (usually takes minutes)
  • Inside you'll find JSON files organized by date
  • Upload the relevant JSON files here

Option B — Health Connect API (For Developers)

  • Android's Health Connect app aggregates data from all health apps
  • You can export specific record types via the Health Connect API
  • For non-dev use: apps like "Health Sync" can bridge Google Fit → CSV

📡 Glucometer / CGM Direct Export

If you use a CGM (Continuous Glucose Monitor)

  • Freestyle Libre (Abbott): Use LibreView (libreview.com) → Reports → Export CSV. LibreLink app also syncs automatically.
  • Dexcom G6/G7: Use Dexcom Clarity app → Export → CSV
  • Manual Glucometer: Log readings in Google Fit or Apple Health, then export as above. Or enter directly on this page.

💡 Recommendation for You

Given your HbA1c of 13.9%, a CGM (Continuous Glucose Monitor) is strongly recommended. The Freestyle Libre 2/3 is available in India (~₹2,500–3,500 per 14-day sensor). It gives you 24/7 glucose data, trend arrows, and alerts for highs/lows. The data can be exported as CSV from LibreView and uploaded directly here.

Even a basic glucometer (Accu-Chek, OneTouch) with 4 readings/day (fasting, pre-lunch, pre-dinner, bedtime) gives enough data to spot patterns and adjust diet/medication. The key is consistency — daily readings for at least 2 weeks to establish a baseline.

🔄 Automated Sync (Future Enhancement)

Planned: GitHub Actions + Google Fit API

A future enhancement could use a GitHub Actions workflow that runs daily, pulls data from Google Fit via API, and auto-updates the data/readings.json file. This would keep the dashboard always up-to-date without manual exports.

Technical approach:

  • Create a Google Cloud project with Fitness API access
  • Store OAuth refresh token as a GitHub Secret
  • A scheduled GitHub Action runs daily, queries the API for new readings
  • Appends new data to data/readings.json and commits
  • GitHub Pages auto-deploys with updated data

This requires initial setup but then runs hands-free. We can build this as a Phase 2 once the manual flow is working.
